这是一个非常典型的 **Linux 目录命名本地化**（中文环境）导致的问题。

从你的 `ls` 输出可以看到，你的家目录下的文件夹名称是中文的（**下载**），而你输入的是英文的 `Downloads`。Linux 的文件系统对大小写敏感，且完全遵循字面值，所以它找不到 `Downloads`。

你有两个方案可以解决：

### 方案 1：直接进入中文目录（最快）
在终端输入 `cd` 后面接中文即可。你可以利用 Tab 键补全，效率很高：
```bash
cd 下[按Tab键]
```
终端会自动帮你补全为 `cd 下载/`。

---

### 方案 2：将目录名改回英文（开发者推荐）
作为 IoT 开发者，以后在终端频繁输入中文目录会非常痛苦（甚至可能遇到脚本路径编码问题）。我建议你把这些标准的家目录名改回英文。

1.  **运行修改命令：**
    在终端输入：
    ```bash
    LC_ALL=C xdg-user-dirs-update --force
    ```

2.  **手动重命名现有目录：**
    上述命令会创建英文文件夹，但不会删除你已经存在的中文文件夹。你需要手动把里面的内容移过去：
    ```bash
    mv 公共/* Public/
    mv 模板/* Templates/
    mv 视频/* Videos/
    mv 图片/* Pictures/
    mv 文档/* Documents/
    mv 下载/* Downloads/
    mv 音乐/* Music/
    mv 桌面/* Desktop/
    ```

3.  **删除空的中文目录：**
    确认文件都挪走后，删掉中文文件夹：
    ```bash
    rmdir 公共 模板 视频 图片 文档 下载 音乐 桌面
    ```

4.  **重启 KDE 桌面：**
    下次登录后，KDE 侧边栏的快捷方式也会自动指向英文路径。

### 为什么会出现这种情况？
这是因为你在安装 Debian 时选择了中文作为系统语言。系统为了对普通用户友好，调用了 `xdg-user-dirs` 插件将 `Downloads` 翻译成了 `下载`。但对于经常在终端（比如你的新 Ghostty）里跳来跳去的开发者来说，英文路径才是“正确姿势”。

**小技巧：** 以后在 Ghostty 里，你可以试着输入 `cd D[Tab]`，如果改成了英文名，它会秒变 `cd Downloads/`。
